# 通配证书 `*.airise.site` 签发与续期（DNS-01）

> 日期：2026-07-19
> 适用：`airise-gateway` 网关托管模式下的 HTTPS 证书。
> 关联：架构 [`260719-nginx部署架构.md`](260719-nginx部署架构.md) §8 · 部署 [`../deploy.md`](../deploy.md) §0 · 新服务接入 [`260719-新服务上线与网关扩展.md`](260719-新服务上线与网关扩展.md)。

## 为什么用通配 + DNS-01

| | 通配 `*.airise.site`（DNS-01） | 单域（HTTP-01） |
|---|---|---|
| 覆盖范围 | 一次签好，**所有子项目零证书成本** | 每加一个子域要单独签一张 |
| 续期占端口 | ❌ 不占（DNS TXT 记录验证） | ✅ 占 80，续期要停网关 |
| 前置依赖 | DNSPod API token + DNS 插件 | 仅 certbot |
| 适用 | 多项目、长期（本架构首选） | 单项目临时跑通 |

> 通配证书 **只能 DNS-01**，无法用 HTTP-01。`*.airise.site` 只匹配一级子域（`pixelpack.`、`www.`、`myapp.`…），**不覆盖裸域 `airise.site`**；裸域若也对外，签发时同时带 `-d airise.site -d '*.airise.site'`。

---

## 1. 准备 DNSPod API 凭证

DNSPod 控制台 → 账号中心 → API 密钥，创建 **DNSPod Token**，得到 `ID` 和 `Token`（注意是 DNSPod 的 token，不是腾讯云 CAM 的 SecretId/SecretKey）。

```bash
sudo mkdir -p /root/.secrets
sudo tee /root/.secrets/dnspod.ini > /dev/null <<'EOF'
dns_dnspod_api_token = <ID>,<TOKEN>
EOF
sudo chmod 600 /root/.secrets/dnspod.ini     # 权限必须 600，否则插件拒绝读
```

## 2. 装 DNS 插件

```bash
sudo pip3 install certbot-dns-dnspod
```

> **Python 3.6 的坑**：服务器上系统 certbot 跑在 Python 3.6（已 EOL），插件很可能装不上或运行报错。两条出路：
> - **优先**：换 snap 版 certbot（自带新 Python）：`sudo snap install --classic certbot && sudo snap set certbot trust-plugin-with-root=ok && sudo snap install certbot-dns-dnspod`。
> - **退一步**：在独立 venv 里 `pip install certbot certbot-dns-dnspod`，不动系统 certbot。
>
> 装好后 `certbot plugins` 应能看到 `dns-dnspod`。

## 3. 签发通配证书

```bash
sudo certbot certonly \
  --dns-dnspod \
  --dns-dnspod-credentials /root/.secrets/dnspod.ini \
  --dns-dnspod-propagation-seconds 30 \
  -d airise.site -d '*.airise.site' \
  --email <你的邮箱> --agree-tos --no-eff-email \
  --server https://acme-v02.api.letsencrypt.org/directory
```

成功标志：`/etc/letsencrypt/live/airise.site/fullchain.pem` 与 `privkey.pem` 生成。
所有站点 conf 的证书路径都指向这个目录（`_template.example` / `pixelpack.airise.site.conf` 默认就是它）。

## 4. 重建网关并验证

conf 路径已匹配，无需改：

```bash
cd /opt/airise-gateway && docker compose up -d      # conf 卷挂载，通常无需 --build
docker exec airise-gateway nginx -t
curl -I https://pixelpack.airise.site          # 应 200（需 pixelpack-web 已在 airise-web 内）
```

浏览器点小锁查看证书，签发域名应含 `*.airise.site`。

## 5. 配自动续期 hook

DNS-01 续期不占端口，证书文件更新后只需让网关 reload。certbot 续期成功后会执行 `/etc/letsencrypt/renewal-hooks/deploy/` 下的脚本：

```bash
sudo tee /etc/letsencrypt/renewal-hooks/deploy/reload-gateway.sh <<'EOF'
#!/bin/sh
# 证书续期后，让网关重新加载（conf 烤进镜像，证书是挂载的，reload 即可生效）
docker exec airise-gateway nginx -s reload
EOF
sudo chmod +x /etc/letsencrypt/renewal-hooks/deploy/reload-gateway.sh

# 验证整个续期链路（不真正续，只跑一遍流程）
sudo certbot renew --dry-run
```

> 通配证书有效期 90 天，certbot 默认在到期前 30 天自动续。`--dry-run` 通过即表示自动续期 + reload hook 链路正常。

## 6. 从单域迁移到通配（本次场景）

服务器原本只有一张单域证书 `www.airise.site`（HTTP-01）。迁移步骤：

1. 按上文签发通配证书（步骤 1–4）。
2. 确认所有引用 `live/www.airise.site` 的 conf 都已切到 `live/airise.site`（本次仅 PixelPack，且其 conf 本就指向 `airise.site`，无需改）。
3. 站点跑稳后，删旧证书：
   ```bash
   sudo certbot delete --cert-name www.airise.site
   ```
4. 若旧证书配的是 HTTP-01 自动续期，删除后 `certbot renew` 不再处理它，无需额外清理。

## 7. 故障速查

- **`The dns_dnspod plugin is not working` / 选项不存在**：插件没装或装的版本与 certbot 不匹配（多见 Python 3.6），换 snap 版 certbot。
- **`Error determining domain identifiers` / DNS 验证超时**：调大 `--dns-dnspod-propagation-seconds`（60–120）；确认 dnspod.ini 里 token 格式是 `ID,TOKEN`、文件权限 600。
- **`unauthorized` / `Incorrect TXT record`**：DNSPod token 权限不足（需该域的 DNS 写权限），或域名 NS 不在 DNSPod。
- **签发成功但浏览器仍报旧证书**：网关没 reload，`docker exec airise-gateway nginx -s reload`；或 CDN/浏览器缓存，换无痕窗口试。
- **`*.airise.site` 报裸域 `airise.site` 不匹配**：通配不覆盖裸域，签发时补 `-d airise.site`。
