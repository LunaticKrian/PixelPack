# Nginx 部署架构（多项目网关模式）

> 状态：实施中
> 日期：2026-07-19
> 域名：`airise.site`（腾讯云 / DNSPod）
> 范围：统一 `airc-nginx` 网关，承载多个前后端分离项目（首个接入：PixelPack）。

---

## 1. 背景与目标

服务器上有多个前后端分离项目（PixelPack、ripro-v5-pro、未来更多），都需要：HTTPS / WSS、文件上传下载与预览、SPA 静态托管。早期 PixelPack 采用「内层 nginx（pixelpack-web）+ 外层网关」的**双重 nginx** 结构，导致 WS/SSE/上传参数要在两层各配一遍，已踩坑（信令 `signal` 卡 pending）。

目标：建立**统一的网关托管模式**，所有项目一致接入，消除内层 nginx。

---

## 2. 现状分析

`airc-nginx` 网关对 ripro 项目**已采用规范的网关直发模式**：

| 挂载（ripro） | 用途 |
|---|---|
| `/opt/ripro-v5-pro/frontend/dist` → `/usr/share/nginx/html` (ro) | ripro SPA，网关直发 |
| `ripro-v5-pro_uploads_data` 卷 → `/var/www/uploads` (ro) | ripro 上传文件，网关直发 |
| `/etc/letsencrypt` → `/etc/letsencrypt` (ro) | certbot 证书 |
| `airc.conf` → `conf.d/airc.conf` (ro) | 单站点配置 |

ripro 模式即目标模式。PixelPack 此前的问题：

1. 🔴 **双重 nginx**：`airc-nginx → pixelpack-web(nginx) → api`，内层多余，WS/SSE 配置需双份。
2. 🟡 容器互联用 `172.17.0.1`（docker0 网桥 IP），脆弱。
3. 🟡 静态/上传走后端（FastAPI StaticFiles），不如 nginx 直发高效。
4. 🟡 多站点 conf.d 是单文件挂载，扩展性差。

---

## 3. 目标架构

```
                    Internet (HTTPS / WSS)
                          │
              ┌───────────▼───────────┐
              │  airc-nginx (网关)     │  唯一入口：TLS 终结 + 路由 + 直发静态
              │  · 通配证书 *.airise.site│
              │  · /opt/nginx/conf.d/  │  多站点目录挂载
              └───────────┬───────────┘
                          │ docker network: airise-web (external, 共享)
            ┌─────────────┼─────────────┬──────────────┐
            ▼             ▼             ▼              ▼
      pixelpack-api   ripro-backend   app3            appN
      (容器名直连，不再暴露宿主机端口、不再用 172.17.0.1)

  共享宿主路径（网关 ro 直发）：
    /opt/<project>/web/dist    → SPA 静态
    /opt/<project>/data/uploads→ 上传 / 预览文件
```

**核心原则**：
- 网关承担 TLS、路由、静态直发；应用容器以**裸服务**接入。
- 所有项目挂在同一外部网络 `airise-web`，网关用**容器名**反代。
- 静态与上传文件放宿主机路径，网关只读挂载、`sendfile` 直发。

## 4. 目录与挂载约定

> 仓库 [`nginx/`](../../nginx/) 目录镜像此结构，含实际配置文件（`conf.d/`、`snippets/`、`README.md`），用 git 同步到服务器 `/opt/nginx/`。

### 网关配置（宿主机）

```
/opt/nginx/
├── conf.d/                         # 多站点（目录挂载进容器）
│   ├── airc.conf                   # ripro（既有，从单文件迁移过来）
│   ├── pixelpack.airise.site.conf  # PixelPack
│   └── _template.conf              # 新项目模板
└── snippets/                       # （可选）可复用片段
    └── proxy-ws.conf
```

### 项目数据（宿主机）

```
/opt/pixelpack/
├── web/dist/        # SPA 构建产物（git pull 后 npm run build 生成）
└── data/
    ├── data.db      # SQLite
    └── uploads/     # 上传文件（api rw 写入，网关 ro 直发）
```

## 5. 关键配置片段

### 5.1 反代通用参数（WS / SSE / 上传）

每个 `proxy_pass` 的 location 都应包含：

```nginx
proxy_http_version 1.1;
map $http_upgrade $connection_upgrade { default upgrade; '' close; }  # 放 http{} 上下文
proxy_set_header Upgrade $http_upgrade;
proxy_set_header Connection $connection_upgrade;
proxy_set_header Host $host;
proxy_set_header X-Real-IP $remote_addr;
proxy_set_header X-Forwarded-For $proxy_add_x_forwarded_for;
proxy_set_header X-Forwarded-Proto $scheme;
proxy_buffering off;          # SSE / WS 必须
proxy_read_timeout 600s;
client_max_body_size 25m;     # 上传上限
```

### 5.2 新项目模板 `conf.d/_template.conf`

```nginx
server {
    listen 443 ssl;
    http2 on;
    server_name <project>.airise.site;

    ssl_certificate     /etc/letsencrypt/live/airise.site/fullchain.pem;   # 通配证书
    ssl_certificate_key /etc/letsencrypt/live/airise.site/privkey.pem;
    ssl_protocols TLSv1.2 TLSv1.3;

    root  /var/www/<project>;
    index index.html;
    client_max_body_size 25m;

    location /api/ {
        proxy_pass http://<project>-api:8000;        # 容器名，需同在 airise-web
        proxy_http_version 1.1;
        proxy_set_header Upgrade $http_upgrade;
        proxy_set_header Connection "upgrade";
        proxy_set_header Host $host;
        proxy_set_header X-Forwarded-For $proxy_add_x_forwarded_for;
        proxy_set_header X-Forwarded-Proto $scheme;
        proxy_buffering off;
        proxy_read_timeout 600s;
    }

    location /uploads/ {
        alias /var/www/<project>-uploads/;
        add_header Content-Disposition "inline";     # 预览：浏览器内联
    }

    location / { try_files $uri $uri/ /index.html; }
}

server {
    listen 80;
    server_name <project>.airise.site;
    return 301 https://$host$request_uri;
}
```

## 6. PixelPack 落地

### 6.1 后端 compose（仓库 `docker-compose.yml`，已改）

api-only，加入 `airise-web` 外部网络，`UPLOAD_DIR=/app/data/uploads` 指向挂载卷。

### 6.2 网关站点 `conf.d/pixelpack.airise.site.conf`

```nginx
server {
    listen 443 ssl;
    http2 on;
    server_name pixelpack.airise.site;

    ssl_certificate     /etc/letsencrypt/live/airise.site/fullchain.pem;

    location /api/ {
        proxy_pass http://pixelpack-api:8000;
        proxy_http_version 1.1;
        proxy_set_header Upgrade $http_upgrade;
        proxy_set_header Connection "upgrade";
        proxy_set_header Host $host;
        proxy_set_header X-Real-IP $remote_addr;
        proxy_set_header X-Forwarded-For $proxy_add_x_forwarded_for;
        proxy_set_header X-Forwarded-Proto $scheme;
        proxy_buffering off;
        proxy_read_timeout 600s;
    }

    location /uploads/ {
        alias /var/www/pixelpack-uploads/;
        add_header Content-Disposition "inline";
    }

    location / { try_files $uri $uri/ /index.html; }
}

server {
    listen 80;
    server_name pixelpack.airise.site;
    return 301 https://$host$request_uri;

```

### 6.3 网关容器挂载（在 airc-nginx compose 上追加）

```yaml
services:
  airc-nginx:
    volumes:
      # ── 既有（ripro）──
      - /opt/ripro-v5-pro/frontend/dist:/usr/share/nginx/html:ro
      - ripro-v5-pro_uploads_data:/var/www/uploads:ro
      - /etc/letsencrypt:/etc/letsencrypt:ro
      # ── conf.d 改为目录挂载 ──
      - /opt/nginx/conf.d:/etc/nginx/conf.d:ro
      # ── 新增（pixelpack）──
      - /opt/pixelpack/web/dist:/var/www/pixelpack:ro
      - /opt/pixelpack/data/uploads:/var/www/pixelpack-uploads:ro
    networks:
      - default        # 既有 ripro 网络
      - airise-web     # 新增共享网络
networks:
  airise-web:
    external: true
```

> 既有 `airc.conf` 单文件挂载需移除，把该文件放进 `/opt/nginx/conf.d/airc.conf`，改用上面的目录挂载。

## 7. 实施步骤

### 7.1 创建共享网络

```bash
docker network create airise-web
```

### 7.2 迁移网关 conf.d 为目录挂载

```bash
mkdir -p /opt/nginx/conf.d
# 把现有 airc.conf 移进去（从原单文件挂载路径复制）
cp /opt/ripro-v5-pro/docker/nginx/airc.conf /opt/nginx/conf.d/airc.conf
# 新建 pixelpack 站点文件（内容见 §6.2）
vi /opt/nginx/conf.d/pixelpack.airise.site.conf
```

### 7.3 更新 airc-nginx compose 并重建

按 §6.3 改 airc-nginx 的 compose，然后：

```bash
docker compose up -d --force-recreate airc-nginx
docker exec airc-nginx nginx -t
```

### 7.4 部署 PixelPack api

```bash
cd /opt/pixelpack
git pull
cp server/.env.example server/.env && vi server/.env   # 填 GLM token
chown -R 1000:1000 data                                 # 非 root 容器用户权限
docker compose up -d --build
```

### 7.5 构建前端 SPA 到宿主机

```bash
cd /opt/pixelpack/web
npm install        # 首次
npm run build      # 产物 → /opt/pixelpack/web/dist，网关直发
```

### 7.6 DNS + 证书

DNS（腾讯云 / DNSPod）：

```
pixelpack.airise.site  →  <服务器公网IP>   (A)
```

证书：见 §8。

## 8. HTTPS 证书

已有 `/etc/letsencrypt`（certbot）。两种方式：

### A. 通配证书 `*.airise.site`（DNS-01，推荐——多项目一劳永逸）

```bash
pip install certbot-dns-dnspod
# /root/dnspod.ini：dns_dnspod_token = <id>,<token>，chmod 600
certbot certonly --dns-dnspod \
  --dns-dnspod-credentials /root/dnspod.ini \
  -d airise.site -d '*.airise.site'
```

所有子域共用 `/etc/letsencrypt/live/airise.site/`，certbot 自动续期。

### B. 单域证书（HTTP-01，免装插件——单项目最快）

```bash
certbot certonly --nginx -d pixelpack.airise.site
# 站点 conf 证书路径改为 /etc/letsencrypt/live/pixelpack.airise.site/
```

> 当前仅接入 PixelPack 一个项目，可先用 B 跑通；加第二个项目时切到 A。

## 9. 新项目接入清单

1. DNS：`<project>.airise.site` A 记录指向服务器。
2. `conf.d/<project>.airise.site.conf`（照抄 `_template.conf`，改 project 名）。
3. 项目 compose：后端容器加入 `airise-web` 外部网络；`/opt/<project>/web/dist` 与 `/opt/<project>/data/uploads` 挂出。
4. airc-nginx compose 追加两条 bind（dist + uploads），`--force-recreate`。
5. 网关 `nginx -s reload`。

## 10. 废弃项

PixelPack 旧的独立部署组件在新模式下**已从仓库删除**：

- `web/Dockerfile`、`web/nginx.conf`、`web/.dockerignore`：原内层 nginx 构建产物，已由网关取代并删除。
- compose 中的 `web` 服务、`PIXELPACK_PORT`：已从 `docker-compose.yml` 移除。
